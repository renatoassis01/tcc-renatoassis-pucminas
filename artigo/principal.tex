\begin{abstract}
\noindent
A infraestrutura como código, também conhecida como IaC, surgiu principalmente com a popularização de metodologias ágeis e computação na nuvem, além do movimento cultural denominado \textit{DevOps}. A \textit{IaC} define como fundamento a automatização de processos das operações de infraestrutura, utilizando uma abordagem para provisionar e gerenciar recursos de computação por meio de codificação ao invés do processo tradicional. Ao tratar a infraestrutura como código, as organizações podem usar as melhores práticas de desenvolvimento de software, incluindo revisão de código, controle de versão, documentação e testes da infraestrutura. Com \textit{Iac} novos conceitos e ferramentas surgem, o que pode ser um dificultador para equipes inexperientes com computação na nuvem. Esta abordagem trás uma necessidade de mudança de paradigmas, além de entender novos conceitos. Para adoção da infraestrutura como código as equipes precisam escolher uma ferramenta ideal que atenda às suas necessidades. Neste artigo é apresentado um estudo comparativo entre ferramentas de IaC e apresenta aspectos positivos e negativo dessas; indicando qual ferramenta se destaca.
\\\textbf{\keyword{Palavras-chave: }}  Infraestrutura como código,Automação, Provisionamento, Computação na nuvem, Terraform, Ansible 
\end{abstract}

\begin{abstract}
\noindent
Infrastructure as Code also known as IaC first emerged with the popularization of agile methodologies, cloud computing and the cultural movement denominated DevOps. IaC's purpose is the automation of infrastructure operations processes using an approach to  provision and manage computational resources via coding instead of  traditional methods.
By treating infrastructure as code organizations can use of software development best practices, including code review and version management, documentation and infrastructure testing. IaC new concepts and tools appear, which can be a challenge for a team that lacks experience with cloud computing because it involves a new approach and a  paradigm change.
To adopt infrastructure as code teams must choose an ideal tool that will fulfill their needs. A comparison between IaC tools is presented in this article listing the positive and negative aspects of those tools.
\\\textbf{\keyword{Keywords: }} Infrastructure as Code, Automation, Provisioning, Cloud Computing, Terraform, Ansible 
\end{abstract}
